# fix 子命令

## 功能说明

根据检查报告中的修复建议，批量修复关联关系问题。

## 前置条件

- 必须先执行 `check` 命令生成报告
- 报告文件 `prd/relations/relations.md` 必须存在

## 工作流程

### 步骤 1：验证报告文件存在

检查 `prd/relations/relations.md` 是否存在。

如果不存在：
> ⚠️ 未找到检查报告。请先执行 `/prd-relations check` 生成报告。

### 步骤 2：读取报告

从报告文件的"五、修复建议汇总"章节提取修复项：

- 5.1 必须修复（高优先级）
- 5.2 建议修复（中优先级）
- 5.3 优化建议（低优先级）

### 步骤 3：选择修复范围并审核确认

展示修复清单，用户选择范围后直接审核确认执行。

```
📋 修复清单（共 {{N}} 项）

【必须修复】高优先级（{{N}} 项）
├── F01: 数据不一致 - 用户ID字段长度不一致
├── F02: 业务规则冲突 - 库存扣减时机矛盾
└── F03: 遗漏关联 - 订单管理未记录与库存管理的关联

【建议修复】中优先级（{{N}} 项）
├── F04: 影响渗透缺失 - 订单管理第3章未说明用户信息来源
└── F05: 双向不一致 - 商品管理未记录被订单管理调用

【优化建议】低优先级（{{N}} 项）
├── F06: 循环依赖 - 建议采用事件驱动解耦
└── F07: 紧耦合 - 考虑定义统一的数据契约

请选择修复范围：
1. ✅ 修复高优先级问题（{{N}} 项）
2. ✅ 修复高优先级 + 中优先级问题（{{N}} 项）
3. ✅ 修复全部问题（{{N}} 项）
4. ✏️ 手动选择修复项
5. ❌ 取消操作
```

用户选择后，立即展示对应的修复方案供确认：

```
📝 已选择修复 {{N}} 项问题

────────────────────────────────────────
修复方案预览：

【F01】用户ID字段长度不一致
├── 涉及：M001_用户管理_prd.md、M003_支付管理_prd.md
└── 操作：统一用户ID为32位UUID格式

【F02】业务规则冲突 - 库存扣减时机
├── 涉及：M002_订单管理_prd.md
└── 操作：明确下单时预扣，支付成功后实扣

【F03】遗漏关联 - 订单与库存
├── 涉及：M002_订单管理_prd.md（第7章）
└── 操作：新增库存管理的关联记录
────────────────────────────────────────

请确认：
✅ 确认执行修复
✏️ 提出修改意见
❌ 取消操作
```

⚠️ **必须暂停，等待用户审核确认！**

### 步骤 4：执行修复

用户确认后，逐项执行修复：

1. 读取目标 PRD 文件
2. 根据修复方案修改对应章节
3. 更新变更日志章节
4. 写入文件

### 步骤 5：更新 CHANGELOG

在 `prd/CHANGELOG.md` 中记录本次修复：

```markdown
## {{CURRENT_DATE}}

### 📄 M002_订单管理_prd.md 🟡更新

修复关联关系问题，补充遗漏的模块集成记录。

#### 变更章节：与其他模块的集成

**变更内容**：新增与库存管理的关联记录，记录下单/取消订单时的库存扣减/恢复逻辑。

---

### 📄 M003_支付管理_prd.md 🟡更新

修复数据字段一致性问题，统一用户ID字段定义。

#### 变更章节：数据字段定义

**变更内容**：用户ID字段长度从64位改为32位，与用户管理模块保持一致。
```

### 步骤 6：完成确认

```
✅ 修复完成：已修复 {{N}} 项问题，修改 {{N}} 个文件

建议：执行 /prd-relations check 重新检查，确认所有问题已修复
```

## 修复类型说明

| 修复类型 | 修复方式 | 影响范围 |
|----------|----------|----------|
| 数据字段不一致 | 统一字段定义，标注字段来源 | 第 4 章 |
| 业务规则冲突 | 统一规则定义，明确适用场景 | 第 7 章 |
| 遗漏关联 | 新增关联记录 | 第 7 章 |
| 双向不一致 | 补充反向关联记录 | 第 7 章 |
| 影响渗透缺失 | 补充跨模块调用/数据来源说明 | 第 3/4/5/6 章 |

## 注意事项

- 修复操作会直接修改 PRD 文件，建议在版本控制下使用
- 修复后建议重新执行 `check` 命令确认问题已解决
- 对于业务规则冲突，需要用户明确选择保留哪个定义
