# 预览原型命令

## 功能说明

启动本地服务器，在浏览器中预览已生成的原型网页。

## 执行流程

### 步骤 1：检查原型文件

检查 `prototype/` 目录是否存在：

```
检查内容：
1. prototype/ 目录是否存在
2. index.html 文件是否存在
3. 必要的资源文件是否存在
```

**如果原型文件不存在：**

```
❌ 未找到原型文件

请先使用 generate 命令生成原型：
1. 调用 prd-to-prototype skill
2. 选择 generate 命令
3. 等待原型生成完成
4. 再次使用 preview 命令预览
```

### 步骤 2：启动本地服务器

使用 Python 或 Node.js 启动简单的 HTTP 服务器：

**Python 方式（推荐）：**

```bash
cd prototype/
python3 -m http.server 8080
```

**Node.js 方式：**

```bash
cd prototype/
npx serve -p 8080
```

**服务器配置：**

- 默认端口：8080
- 如果端口被占用，自动尝试下一个端口（8081、8082...）
- 支持热重载（可选）

### 步骤 3：打开浏览器

自动打开默认浏览器：

```
URL: http://localhost:8080
```

### 步骤 4：显示预览信息

```
🌐 原型预览服务器已启动

📍 本地地址：http://localhost:8080
📁 原型目录：prototype/

💡 提示：
- 按 Ctrl+C 停止服务器
- 修改原型文件后刷新浏览器即可看到更新
- 支持在手机上访问（同一局域网）
```

## 交互功能

### 端口选择

如果默认端口被占用：

```
⚠️ 端口 8080 已被占用

正在尝试端口 8081...
✅ 服务器已启动在端口 8081

📍 本地地址：http://localhost:8081
```

### 局域网访问

显示局域网访问地址：

```
📱 局域网访问地址：
http://192.168.1.100:8080

您可以在同一局域网的手机或平板上访问此地址预览原型。
```

### 热重载（可选）

如果支持热重载：

```
🔄 热重载已启用

修改原型文件后，浏览器会自动刷新。
```

## 命令选项

### 指定端口

```
preview --port 3000
```

使用指定端口启动服务器。

### 指定目录

```
preview --dir ./my-prototype
```

预览指定目录下的原型文件。

### 禁用自动打开浏览器

```
preview --no-open
```

启动服务器但不自动打开浏览器。

## 错误处理

### 原型文件不存在

```
❌ 错误：未找到原型文件

请先使用 generate 命令生成原型。
```

### 端口全部被占用

```
❌ 错误：无法找到可用端口

尝试的端口：8080-8090 都已被占用。

解决方案：
1. 关闭占用端口的程序
2. 使用 --port 参数指定其他端口
```

### 服务器启动失败

```
❌ 错误：服务器启动失败

原因：[具体错误信息]

解决方案：
1. 检查 Python 或 Node.js 是否正确安装
2. 检查 prototype/ 目录权限
3. 尝试手动启动服务器
```

## 注意事项

1. **服务器运行**：预览期间服务器会持续运行，按 Ctrl+C 停止
2. **文件更新**：修改原型文件后刷新浏览器即可看到更新
3. **移动端预览**：支持在同一局域网的移动设备上预览
4. **端口占用**：如果默认端口被占用，会自动尝试其他端口
