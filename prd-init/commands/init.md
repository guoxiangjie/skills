# init 子命令

## 功能说明

初始化 PRD 文档包结构，创建完整的目录和文件。

## 工作流程

### 步骤1：询问生成位置
使用 AskUserQuestion 工具询问用户要在哪个位置生成 PRD 文档包。

### 步骤2：扫描是否存在可用的 PRD 文档包
- 递归扫描当前路径下的所有文件夹
- 查找是否存在有效的 PRD 结构：
  - 存在 `prd/` 文件夹
  - 文件夹内包含 `README.md`、`PRD_INDEX.md`、`CHANGELOG.md` 三个文件
- 如果找到有效的 PRD 结构：
  - 提示用户已检测到可用的 PRD 文档包及其位置
  - 询问是否使用现有的 PRD 结构（跳过初始化）或选择其他位置重新初始化
- 如果未找到，继续执行初始化流程

### 步骤3：创建目录结构
在目标位置创建以下目录和文件：

```
目标位置/
└── prd/
    ├── README.md              # 项目说明文档
    ├── PRD_INDEX.md           # PRD模块总集索引
    ├── CHANGELOG.md           # 版本变更日志
    └── modules/               # 功能模块PRD目录（空文件夹）
```

### 步骤4：生成文件内容
使用模板文件生成各个文档的内容，替换变量。

### 步骤5：确认完成
向用户确认 PRD 文档包已成功创建，并提示下一步可以使用 `prd-suite` skill 来生成具体内容。

## 注意事项

- 如果目标位置不存在，会自动创建
- 所有文件使用 UTF-8 编码
- `{{CURRENT_DATE}}` 替换为当前日期，格式为 YYYY-MM-DD
- `{{PROJECT_NAME}}` 替换为目标文件夹的名称
