# create 子命令

## 功能说明

创建新的 PRD 文档，支持创建主PRD和模块PRD。

## 子命令

### create main
- **功能**：创建主PRD文档
- **描述**：生成完整的主PRD文档，包含项目背景、目标、核心业务流程等
- **支持**：自然语言需求模板快速生成

### create module
- **功能**：创建模块PRD文档
- **描述**：生成指定模块的PRD文档，包含详细功能需求、用户故事等
- **支持**：智能推断机制，减少重复输入

## 工作流程

### 创建主PRD流程

1. **XMind MCP 检测**：检测 XMind MCP 服务是否可用
2. **需求模板检测**：检查是否存在 `requirement_template.md` 文件
3. **模式选择**：
   - 使用 XMind 文件：通过 XMind MCP 读取 xmind 文件生成 PRD（推荐，需要 XMind MCP）
   - 使用需求文件：直接从自然语言需求生成 PRD
   - 混合模式：需求文件 + 交互补充
   - 手动创建：使用交互式流程
4. **信息收集**：根据选择的模式收集必要信息
5. **生成文档**：根据收集的信息生成完整 PRD
6. **审核确认**：用户审核生成的内容
7. **完成创建**：创建 `{项目名称}_main_prd.md` 文件

## 使用 XMind 文件创建主 PRD 详细流程

### 前置条件

- XMind MCP 服务必须可用
- 当前工作目录下存在 .xmind 文件

### 流程步骤

1. **检测 XMind MCP**
   - 尝试调用 `mcp_XMind_list_xmind_directory` 工具
   - 如果失败，提示用户：
     > ⚠️ 未检测到 XMind MCP 服务，无法使用 XMind 文件功能。
     > 请选择其他创建模式：使用需求文件、混合模式或手动创建。

2. **扫描 XMind 文件**
   - 调用 `mcp_XMind_list_xmind_directory` 扫描当前工作目录
   - 获取所有 .xmind 文件列表

3. **展示文件列表**
   - 如果没有找到 xmind 文件：
     > 📂 当前目录下未找到 .xmind 文件。
     > 请选择其他创建模式，或将 xmind 文件放置到当前目录。
   - 如果找到文件，展示列表供用户选择

4. **用户选择文件**
   - 展示文件列表，格式如：
     ```
     找到以下 XMind 文件：
     1. 需求文档.xmind
     2. 产品规划.xmind
     3. 功能设计.xmind
     
     请选择要使用的文件（输入序号）：
     ```
   - 用户输入序号选择文件

5. **解析 XMind 内容**
   - 调用 `mcp_XMind_read_xmind` 读取选中的文件
   - 提取完整的思维导图结构，包括：
     - 根主题和所有子主题
     - 节点标题、备注、标签
     - 节点关系和连接
     - 任务状态和优先级

6. **生成 XMind 内容文档**
   - 创建 `xmind_content.md` 文件
   - 文档结构：
     ```markdown
     # XMind 内容解析文档
     
     ## 文档基础信息
     - **源文件**：{文件名}.xmind
     - **解析时间**：{时间}
     - **根主题**：{根主题标题}
     
     ## 思维导图结构
     
     {层级展示所有节点}
     
     ## 节点详情
     
     {每个节点的详细信息}
     
     ## 关系和连接
     
     {节点之间的关系}
     ```
   - ⚠️ **此时暂停，展示内容文档供用户确认**

7. **确认内容文档**
   - 展示生成的 `xmind_content.md` 内容
   - 提供审核选项：
     - ✅ **确认继续**：继续生成主 PRD
     - ✏️ **提出修改意见**：修改内容文档
     - 🔄 **重新选择文件**：返回步骤 4 重新选择
     - ❌ **取消操作**：终止流程

8. **提取 PRD 信息**
   - 从 `xmind_content.md` 中提取项目信息：
     - 项目名称和背景
     - 项目目标
     - 功能模块
     - 用户角色
     - 业务流程
     - 其他相关信息

9. **生成主 PRD**
   - 严格按照 `templates/main_prd_template.md` 模板格式
   - 根据提取的信息生成完整的主 PRD 内容
   - ⚠️ **此时不创建文件，只生成内容**

10. **展示 PRD 内容**
    - 展示完整的主 PRD 内容供用户查看

11. **暂停等待审核**
    - ⚠️ **必须暂停，等待用户审核确认**
    - **不能自动继续执行后续操作**
    - 提供审核选项：
      - ✅ **确认通过**：继续执行后续操作
      - ✏️ **提出修改意见**：用户输入修改意见，AI 修改后再次展示审核
      - 🔄 **重新生成**：重新生成内容，再次展示审核
      - ❌ **取消操作**：终止流程，不创建文件

12. **审核确认**：用户确认内容无误后，继续执行

13. **完成创建**：创建主 PRD 文件并更新相关索引

### XMind 内容提取规则

从 XMind 思维导图中提取 PRD 信息时，遵循以下规则：

1. **项目名称**：从根主题或第一层子主题提取
2. **项目背景**：查找包含"背景"、"简介"、"概述"等关键词的节点
3. **项目目标**：查找包含"目标"、"目的"、"愿景"等关键词的节点
4. **功能模块**：查找包含"功能"、"模块"、"特性"等关键词的节点及其子节点
5. **用户角色**：查找包含"用户"、"角色"、"参与者"等关键词的节点
6. **业务流程**：查找包含"流程"、"步骤"、"过程"等关键词的节点
7. **非功能需求**：查找包含"性能"、"安全"、"可用性"等关键词的节点
8. **验收标准**：查找包含"验收"、"标准"、"条件"等关键词的节点

### 适用场景

- 已有 XMind 格式的需求文档
- 需求以思维导图形式整理
- 需要从产品规划文档转换 PRD
- 团队使用 XMind 进行需求管理

### 注意事项

- XMind MCP 不可用时，自动隐藏此选项
- 解析后的内容文档需要用户确认
- 生成 PRD 前需要审核确认
- 保持与模板格式一致

### 创建模块PRD流程

1. **前置条件**：检查主PRD是否存在
2. **选择模块**：从主PRD的模块清单中选择要创建的模块
3. **模式选择**：
   - **快速模式**：根据主PRD和模块信息自动生成完整内容
   - **问答模式**：逐章节询问核心业务信息并生成内容
4. **内容生成**：根据选择的模式生成模块PRD内容
5. **审核确认**：用户审核生成的内容
6. **完成创建**：创建 `modules/{模块编号}_{模块名称}_prd.md` 文件

## 自然语言需求模板支持

支持使用 `requirement_template.md` 文件快速创建PRD：

1. **模板内容**：用自然语言描述项目需求
2. **智能提取**：AI自动从自然语言中提取结构化信息
3. **快速生成**：1轮完成PRD生成

## 模块 PRD 快速模式详细流程

### 流程步骤

1. **选择模块**
   - 从主 PRD 的模块清单中选择要创建的模块
   - 获取模块编号、名称、描述、优先级等基础信息

2. **提取主PRD信息**
   - 从主 PRD 中提取相关信息：
     - 项目背景和目标
     - 用户角色和权限
     - 相关模块信息
     - 非功能需求
     - 验收标准

3. **自动生成完整内容**
   - 根据主 PRD 信息和模块信息
   - 严格按照 `templates/module_prd_template.md` 模板格式
   - 自动填充所有章节内容
   - ⚠️ **此时不要创建文件，只生成内容**

4. **展示内容**
   - 展示完整的模块 PRD 内容供用户查看

5. **暂停等待审核**
   - ⚠️ **必须暂停，等待用户审核确认**
   - **不能自动继续执行后续操作**
   - 提供审核选项：
     - ✅ **确认通过**：继续执行后续操作
     - ✏️ **提出修改意见**：用户输入修改意见，AI 修改后再次展示审核
     - 🔄 **重新生成**：重新生成内容，再次展示审核
     - ❌ **取消操作**：终止流程，不创建文件

6. **审核确认**：用户确认内容无误后，继续执行

7. **完成创建**：创建模块 PRD 文件并更新相关索引

### 适用场景

- 主 PRD 信息完整详细
- 模块功能相对标准
- 需要快速生成 PRD
- 对模块有基本了解

## 问答模式详细流程

### 流程步骤

1. **选择模块**
   - 从主 PRD 的模块清单中选择要创建的模块
   - 获取模块编号、名称、描述、优先级等基础信息

2. **逐章节询问和生成**
   按照模块 PRD 模板的章节顺序，依次进行：

   **第 1 章：文档基础信息**（自动填充）
   - 从主 PRD 和模块信息自动提取
   - 无需询问

   **第 2 章：模块背景与目标**
   - **询问内容**：
     - 模块背景：描述此功能模块的业务背景
     - 模块目标：列出模块的主要目标
   - **参考信息**：
     - 主 PRD 的项目背景和目标
     - 模块在主 PRD 中的描述
   - **生成内容**：根据用户输入生成该章节
   - **章节审核**：展示内容，提供审核选项

   **第 3 章：详细功能需求**
   - **询问内容**：
     - 功能清单：列出所有功能点
     - 功能流程：描述主要功能流程
     - 功能详细说明：详细描述每个功能点
   - **参考信息**：
     - 主 PRD 的功能模块清单
     - 模块在主 PRD 中的描述
   - **生成内容**：根据用户输入生成该章节
   - **章节审核**：展示内容，提供审核选项

   **第 4 章：用户故事详述**
   - **询问内容**：
     - 用户故事列表：列出所有用户故事
   - **参考信息**：
     - 主 PRD 的用户角色和权限
     - 功能需求章节的内容
   - **生成内容**：根据用户输入生成该章节
   - **章节审核**：展示内容，提供审核选项

   **第 5 章：用户界面设计**
   - **询问内容**：
     - 核心屏幕和视图：列出主要页面
     - 页面流程：描述页面之间的导航
     - 页面说明：详细描述每个页面
   - **参考信息**：
     - 功能需求章节的内容
     - 用户故事章节的内容
   - **生成内容**：根据用户输入生成该章节
   - **章节审核**：展示内容，提供审核选项

   **第 6 章：业务规则与边界条件**
   - **询问内容**：
     - 业务规则：列出所有业务规则
     - 边界条件：描述边界和限制
   - **参考信息**：
     - 功能需求章节的内容
     - 用户故事章节的内容
   - **生成内容**：根据用户输入生成该章节
   - **章节审核**：展示内容，提供审核选项

   **第 7 章：与其他模块的集成**
   - **询问内容**：
     - 集成关系：列出与其他模块的集成
     - 集成流程：描述集成流程
   - **参考信息**：
     - 主 PRD 的模块关系图
     - 主 PRD 的模块清单
   - **生成内容**：根据用户输入生成该章节
   - **章节审核**：展示内容，提供审核选项

   **第 8 章：异常与错误处理**
   - **询问内容**：
     - 错误状态：描述可能的错误状态
     - 错误码：列出错误码和错误信息
   - **参考信息**：
     - 功能需求章节的内容
     - 业务规则章节的内容
   - **生成内容**：根据用户输入生成该章节
   - **章节审核**：展示内容，提供审核选项

   **第 9 章：埋点与数据指标**
   - **询问内容**：
     - 埋点位置：列出需要埋点的位置
     - 事件名和上报字段：描述埋点详情
   - **参考信息**：
     - 用户故事章节的内容
     - 功能需求章节的内容
   - **生成内容**：根据用户输入生成该章节
   - **章节审核**：展示内容，提供审核选项

   **第 10 章：模块级验收标准**
   - **询问内容**：
     - 功能验收：列出功能验收标准
     - 界面验收：列出界面验收标准
     - 集成验收：列出集成验收标准
   - **参考信息**：
     - 主 PRD 的验收标准
     - 功能需求章节的内容
   - **生成内容**：根据用户输入生成该章节
   - **章节审核**：展示内容，提供审核选项

   **第 11 章：附录**（可选）
   - **询问内容**：
     - 术语定义：列出专业术语
     - UI 设计图：提供设计图链接
   - **参考信息**：
     - 主 PRD 的附录
   - **生成内容**：根据用户输入生成该章节
   - **章节审核**：展示内容，提供审核选项

3. **完整审核**
   - 展示完整的模块 PRD 内容
   - ⚠️ **必须暂停，等待用户审核确认**
   - **不能自动继续执行后续操作**
   - 提供审核选项：
     - ✅ **确认通过**：继续执行后续操作
     - ✏️ **提出修改意见**：用户输入修改意见，AI 修改后再次展示审核
     - 🔄 **重新生成特定章节**：选择章节重新生成
     - ❌ **取消操作**：终止流程，不创建文件

4. **审核确认**：用户确认内容无误后，继续执行

5. **完成创建**：创建模块 PRD 文件并更新相关索引

### 章节审核选项

每个章节生成后，提供以下审核选项：

- ✅ **确认通过**：继续下一章节
- ✏️ **提出修改意见**：用户输入修改意见，AI 修改后再次审核该章节
- 🔄 **重新生成**：重新生成该章节内容
- ⏭️ **跳过章节**：跳过当前章节（仅限可选章节）

### 适用场景

- 需要深入思考模块细节
- 主 PRD 信息不够详细
- 模块功能复杂
- 需要引导式填写

### ⚠️ 格式要求

无论使用哪种模式生成 PRD，都必须：

- **严格遵循对应的模板文件格式**（`main_prd_template.md` 或 `module_prd_template.md`）
- **保持章节结构完整**：不遗漏任何必需章节，不添加额外章节
- **保持格式一致**：表格、图表、标题层级必须与模板一致
- **完整替换变量**：所有 `{{变量名}}` 必须替换为实际内容
- **生成后验证**：检查格式是否符合模板要求，发现问题立即修正

## ⚠️ 审核确认机制

**所有 PRD 创建操作都必须经过用户审核确认！**

### 审核流程

1. **生成内容**：AI 生成 PRD 内容（⚠️ 不创建文件）
2. **展示内容**：展示完整内容供用户查看
3. **暂停等待**：⚠️ **必须暂停，等待用户审核**
4. **用户选择**：
   - ✅ **确认通过**：继续执行后续操作
   - ✏️ **提出修改意见**：用户输入修改意见，AI 修改后再次展示审核
   - 🔄 **重新生成**：重新生成内容，再次展示审核
   - ❌ **取消操作**：终止流程，不创建文件
5. **执行后续**：审核通过后才执行后续操作（创建文件、更新索引等）

### 重要原则

- **先审核后执行**：审核通过前不得创建文件
- **必须暂停等待**：AI 不得自动跳过审核步骤
- **支持迭代修改**：用户可多次提出修改意见
- **提供完整选项**：必须提供确认、修改、重新生成、取消四个选项

## 注意事项

- 创建主PRD时，会更新 `CHANGELOG.md`
- 创建模块PRD时，会更新 `PRD_INDEX.md` 和 `CHANGELOG.md`
- 支持版本管理和状态管理
