# create 子命令

## 功能说明

创建新的 PRD 文档，支持创建主PRD和模块PRD。

## 子命令

### create main
- **功能**：创建主PRD文档
- **描述**：生成完整的主PRD文档，包含项目背景、目标、核心业务流程等
- **支持**：自然语言需求模板快速生成

### create module
- **功能**：创建模块PRD文档
- **描述**：生成指定模块的PRD文档，包含详细功能需求、用户故事等
- **支持**：智能推断机制，减少重复输入

## 工作流程

### 创建主PRD流程

1. **模式选择**：先让用户选择创建模式
   - 使用 XMind 文件：通过 XMind MCP 读取 xmind 文件生成 PRD（推荐）
   - 使用需求模板：根据需求模板文件生成 PRD
   - 混合模式：需求模板 + 交互补充
   - 手动创建：使用交互式流程

2. **条件验证**：根据选择的模式验证前置条件
   - XMind 模式：验证 XMind MCP 服务是否可用
   - 需求模板模式：检测项目目录是否有 requirement_template.md
   - 混合模式：检测项目目录是否有 requirement_template.md
   - 手动创建：无需验证，直接开始

3. **信息收集**：根据选择的模式收集必要信息
4. **生成文档**：根据收集的信息生成完整 PRD
5. **审核确认**：用户审核生成的内容
6. **完成创建**：创建 `{项目名称}_main_prd.md` 文件

## 使用 XMind 文件创建主 PRD 详细流程

### 前置条件

- XMind MCP 服务必须可用

### 流程步骤

1. **验证 XMind MCP 服务**
   - 尝试调用 `mcp_XMind_list_xmind_directory` 工具
   - 如果失败，提示用户：
     > ⚠️ 未检测到 XMind MCP 服务，无法使用 XMind 文件功能。
     > 请选择其他创建模式：使用需求模板、混合模式或手动创建。
   - 如果成功，继续下一步

2. **扫描 XMind MCP 配置目录**
   - 调用 `mcp_XMind_list_xmind_directory` 扫描 XMind MCP 配置目录
   - 获取所有 .xmind 文件列表

3. **展示文件列表**
   - 如果没有找到 xmind 文件：
     > 📂 XMind MCP 配置目录下未找到 .xmind 文件。
     > 请选择其他创建模式，或将 xmind 文件放置到 XMind MCP 配置目录。
   - 如果找到文件，展示列表供用户选择

4. **用户选择文件**
   - 展示文件列表，格式如：
     ```
     找到以下 XMind 文件：
     1. 需求文档.xmind
     2. 产品规划.xmind
     3. 功能设计.xmind
     
     请选择要使用的文件（输入序号）：
     ```
   - 用户输入序号选择文件

5. **解析 XMind 内容**
   - 调用 `mcp_XMind_read_xmind` 读取选中的文件
   - 提取完整的思维导图结构，包括：
     - 根主题和所有子主题
     - 节点标题、备注、标签
     - 节点关系和连接
     - 任务状态和优先级

6. **生成 XMind 内容文档**
   - 参考 `templates/xmind_content_template.md` 模板结构生成内容
   - **生成要求**：
     - **第一部分：原始内容解析**
       - 完整列出所有节点的层级结构
       - 详细列出每个节点的完整信息（标题、层级、备注、标签、链接、子节点）
       - 不遗漏任何节点内容
     - **第二部分：内容分析与总结**
       - 基于全部节点内容，经过分析思考，总结以下关键信息：
         - 功能概述：产品/项目的核心功能是什么，解决什么问题
         - 使用人（用户角色）：目标用户是谁，有哪些角色
         - 目标：产品/项目的目标是什么
         - 功能模块列表：整理出功能模块（⚠️ 此时先不生成，等待层级选择）
         - 业务流程：总结核心业务流程
     - **不显示节点来源**：总结内容不标注"从哪个节点提取"
   - ⚠️ **此时暂停，展示内容文档供用户确认**

7. **确认内容文档**
   - 展示生成的 XMind 内容文档（不含功能模块列表）
   - ⚠️ **必须暂停，等待用户审核确认**
   - **不能自动继续执行后续操作**
   - 提供审核选项：
     - ✅ **确认继续**：继续下一步，进入功能模块层级选择
     - ✏️ **提出修改意见**：用户输入修改意见，AI 修改后再次展示审核
     - 🔄 **重新选择文件**：返回步骤 4 重新选择
     - ❌ **取消操作**：终止流程

7.5 **功能模块层级选择**（新增步骤，在确认内容文档之后）
   - 展示 XMind 层级结构分析和 AI 推荐
   - 提供三种选择方式：
     - **选项 1**：确认使用 AI 推荐层级
     - **选项 2**：按固定层级提取（用户输入层级数字）
     - **选项 3**：手动指定功能模块清单

   **层级分析展示格式**：
   ```
   ═══════════════════════════════════════════
   📊 XMind 结构分析

   根主题：{产品名称}

   层级结构预览：
   ├── 第 1 层（共 X 个节点）
   │   ├── 节点 1、节点 2、节点 3...
   ├── 第 2 层（共 X 个节点）
   │   ├── 用户管理、订单管理、支付管理...
   └── 第 3 层（共 X 个节点）
       ├── 用户列表、用户详情、创建用户...

   🤖 AI 分析推荐
   推荐层级：第 X 层
   理由：
   - 节点数量适中（X 个），符合模块划分粒度
   - 节点命名清晰，符合功能模块命名规范
   - 与主 PRD 功能模块清单的颗粒度匹配

   推荐的功能模块清单：
   1. 模块 1
   2. 模块 2
   3. 模块 3
   ...
   ═══════════════════════════════════════════

   请选择功能模块的提取层级：
   1️⃣ 确认使用 AI 推荐（第 X 层，X 个模块）
   2️⃣ 按固定层级提取（输入层级数字 1/2/3/4/5）
   3️⃣ 手动指定功能模块清单
   ```

   **选项 1：确认 AI 推荐**
   - 直接使用推荐的层级
   - 提取该层所有节点作为功能模块清单
   - 继续后续流程

   **选项 2：按固定层级提取**
   ```
   请输入层级数字（1-5）：
   > 用户输入

   已选择：第 X 层（共 X 个节点）
   提取的功能模块：
   1. 模块 1
   2. 模块 2
   ...

   确认是否继续？[确认/重新选择]
   ```

   **选项 3：手动指定功能模块清单**
   ```
   请手动输入功能模块清单（每行一个，格式：编号 | 名称 | 描述 | 优先级）：

   示例：
   M-01 | 用户管理 | 负责用户相关功能 | 高
   M-02 | 订单管理 | 负责订单处理功能 | 高

   输入完成后，发送"完成"结束：
   > 用户输入...
   ```

   - 用户输入完成后，展示确认列表
   - 用户确认后，继续后续流程

   **层级选择完成后的处理**：
   - 将选定的功能模块清单填充到 `xmind_content.md` 的"2.4 功能模块列表"章节
   - 生成格式化的模块表格（编号、名称、描述、优先级）
   - 用户确认填充结果后，继续后续流程

8. **提取 PRD 信息**
   - 从 `xmind_content.md` 的"内容分析与总结"部分提取项目信息：
     - 功能概述
     - 用户角色
     - 项目目标
     - 功能模块（使用层级选择步骤中确定的模块清单）
     - 业务流程

9. **生成主 PRD**
   - 严格按照 `templates/main_prd_template.md` 模板格式
   - 根据提取的信息生成完整的主 PRD 内容
   - ⚠️ **此时不创建文件，只生成内容**

10. **展示 PRD 内容**
    - 展示完整的主 PRD 内容供用户查看

11. **暂停等待审核**
    - ⚠️ **必须暂停，等待用户审核确认**
    - **不能自动继续执行后续操作**
    - 提供审核选项：
      - ✅ **确认通过**：继续执行后续操作
      - ✏️ **提出修改意见**：用户输入修改意见，AI 修改后再次展示审核
      - 🔄 **重新生成**：重新生成内容，再次展示审核
      - ❌ **取消操作**：终止流程，不创建文件

12. **审核确认**：用户确认内容无误后，继续执行

13. **完成创建**：创建主 PRD 文件并更新相关索引

### XMind 内容提取规则

从 XMind 思维导图中提取 PRD 信息时，遵循以下原则：

1. **整体思考总结**：不是机械地按关键词提取，而是先完整解析所有节点内容，再综合分析思考，总结关键信息
2. **提取内容**：
   - 功能概述：产品/项目的核心功能是什么，解决什么问题
   - 使用人（用户角色）：目标用户是谁，有哪些角色
   - 目标：产品/项目的目标是什么
   - 功能模块列表：整理出功能模块
   - 业务流程：总结核心业务流程
3. **不提取内容**：
   - 非功能需求
   - 验收标准
   - 项目背景
4. **不显示节点来源**：总结内容不标注"从哪个节点提取"

### 适用场景

- 已有 XMind 格式的需求文档
- 需求以思维导图形式整理
- 需要从产品规划文档转换 PRD
- 团队使用 XMind 进行需求管理

### 注意事项

- XMind MCP 不可用时，自动隐藏此选项
- 解析后的内容文档需要用户确认
- 生成 PRD 前需要审核确认
- 保持与模板格式一致

## 使用需求模板创建主 PRD 详细流程

### 流程步骤

1. **检测需求模板文件**
   - 检查项目目录是否存在 `requirement_template.md`
   - 如果不存在：
     - 从 `templates/requirement_template.md` 复制模板到项目目录
     - 提示用户：
       > 📝 已创建需求模板文件 `requirement_template.md`。
       > 请填写模板内容后告知我继续。
     - 等待用户确认填写完成
   - 如果存在，继续下一步

2. **读取需求模板内容**
   - 读取项目目录下的 `requirement_template.md`
   - 解析自然语言需求

3. **提取 PRD 信息**
   - 从需求模板中提取项目信息：
     - 项目名称和背景
     - 项目目标
     - 功能模块
     - 用户角色
     - 业务流程
     - 其他相关信息

4. **生成主 PRD**
   - 严格按照 `templates/main_prd_template.md` 模板格式
   - 根据提取的信息生成完整的主 PRD 内容
   - ⚠️ **此时不创建文件，只生成内容**

5. **展示 PRD 内容**
   - 展示完整的主 PRD 内容供用户查看

6. **暂停等待审核**
   - ⚠️ **必须暂停，等待用户审核确认**
   - **不能自动继续执行后续操作**
   - 提供审核选项：
     - ✅ **确认通过**：继续执行后续操作
     - ✏️ **提出修改意见**：用户输入修改意见，AI 修改后再次展示审核
     - 🔄 **重新生成**：重新生成内容，再次展示审核
     - ❌ **取消操作**：终止流程，不创建文件

7. **审核确认**：用户确认内容无误后，继续执行

8. **完成创建**：创建主 PRD 文件并更新相关索引

### 适用场景

- 已有需求文档或需求描述
- 需求以自然语言形式整理
- 需要快速生成 PRD
- 需求内容相对完整

### 注意事项

- 需求模板需要用户填写完整
- 生成 PRD 前需要审核确认
- 保持与模板格式一致

### 创建模块PRD流程

1. **前置条件**：检查主PRD是否存在
2. **选择模块**：从主PRD的模块清单中选择要创建的模块
3. **模式选择**：
   - **快速模式**：根据主PRD和模块信息自动生成完整内容
   - **问答模式**：逐章节询问核心业务信息并生成内容
4. **内容生成**：根据选择的模式生成模块PRD内容
5. **审核确认**：用户审核生成的内容
6. **完成创建**：创建 `modules/{模块编号}_{模块名称}_prd.md` 文件

## 自然语言需求模板支持

支持使用 `requirement_template.md` 文件快速创建PRD：

1. **模板内容**：用自然语言描述项目需求
2. **智能提取**：AI自动从自然语言中提取结构化信息
3. **快速生成**：1轮完成PRD生成

### ⚠️ 格式要求

- **保持章节结构完整**：不遗漏任何必需章节，不添加额外章节
- **保持格式一致**：表格、图表、标题层级必须与模板一致
- **完整替换变量**：所有 `{{变量名}}` 必须替换为实际内容
- **生成后验证**：检查格式是否符合模板要求，发现问题立即修正

## ⚠️ 审核确认机制

**所有 PRD 创建操作都必须经过用户审核确认！**

### 审核流程

1. **生成内容**：AI 生成 PRD 内容（⚠️ 不创建文件）
2. **展示内容**：展示完整内容供用户查看
3. **暂停等待**：⚠️ **必须暂停，等待用户审核**
4. **用户选择**：
   - ✅ **确认通过**：继续执行后续操作
   - ✏️ **提出修改意见**：用户输入修改意见，AI 修改后再次展示审核
   - 🔄 **重新生成**：重新生成内容，再次展示审核
   - ❌ **取消操作**：终止流程，不创建文件
5. **执行后续**：审核通过后才执行后续操作（创建文件、更新索引等）

### 重要原则

- **先审核后执行**：审核通过前不得创建文件
- **必须暂停等待**：AI 不得自动跳过审核步骤
- **支持迭代修改**：用户可多次提出修改意见
- **提供完整选项**：必须提供确认、修改、重新生成、取消四个选项
