# update 子命令

## 功能说明

更新现有的 PRD 文档，支持更新主PRD和模块PRD。

## 子命令

### update main
- **功能**：更新主PRD文档
- **描述**：修改现有主PRD的内容，支持单章节更新
- **支持**：版本管理和变更记录

### update module
- **功能**：更新模块PRD文档
- **描述**：修改现有模块PRD的内容，支持单章节更新
- **支持**：版本管理和变更记录

## 工作流程

### 更新PRD流程

1. **选择文档**：选择要更新的PRD文档（主PRD或模块PRD）
2. **选择章节**：选择要更新的章节
3. **信息收集**：收集该章节的更新信息
4. **生成内容**：生成更新后的章节内容
   - ⚠️ **此时不要更新文件，只生成内容**
   - 必须严格按照对应的模板格式生成
   - 保持章节结构、表格格式、Mermaid 图表格式与模板一致
   - 确保与现有文档格式保持一致
5. **展示内容**：将更新后的章节内容展示给用户查看
6. **暂停等待审核**：
   - ⚠️ **必须暂停，等待用户审核确认**
   - **不能自动继续执行后续操作**
   - 提供审核选项：
     - ✅ **确认通过**：继续执行后续操作
     - ✏️ **提出修改意见**：用户输入修改意见，AI 修改后再次展示审核
     - 🔄 **重新生成**：重新生成内容，再次展示审核
     - ❌ **取消操作**：终止流程，不更新文件
7. **审核确认**：用户确认内容无误后，继续执行
8. **版本管理**：询问是否需要升级版本号
9. **完成更新**：更新PRD文档和相关文件

## 版本管理

### 版本升级流程
1. **确定变更类型**：重大变更或功能变更/修复
2. **计算新版本号**：根据变更类型更新版本号
3. **更新文档**：在PRD文档中更新版本号和变更日志

### 同版本更新流程
1. **更新状态**：将状态从"已评审"改为"未评审"
2. **添加变更记录**：在当前版本下添加更新说明

## 变更记录

### CHANGELOG 记录原则

**重要**：CHANGELOG 本身没有版本概念，它只是一个变更记录文件。

- CHANGELOG 按日期记录变更
- 每条变更记录关联到具体的 PRD 文件及其版本
- PRD 文件有自己的版本号（如 v1.0、v1.1）

### CHANGELOG 记录格式

更新 PRD 时，必须在 CHANGELOG.md 中记录详细的变更信息：

#### 1. 日期标题

```markdown
## 2024-01-15
```

- 使用 `YYYY-MM-DD` 格式
- 按时间倒序排列（最新的在上面）

#### 2. PRD 文件标识

```markdown
### 📄 文件名 (旧版本 → 新版本)
```

- 更新文件：`### 📄 电商平台_main_prd.md (v1.0 → v1.1)`
- 创建文件：`### 📄 modules/001_用户管理_prd.md (创建 v1.0)`
- 删除文件：`### 📄 modules/003_废弃功能_prd.md (删除 v1.2)`

#### 3. 变更概述

```markdown
**变更概述**：简要描述本次对该文件的变更内容
```

#### 4. 章节标识

```markdown
#### 第X章 章节名称
```

- 多个章节变更时，每个章节单独列出
- 创建新文档时使用 `#### 全部章节`

#### 5. 变更详情表格

```markdown
| 变更项 | 变更前 | 变更后 | 变更原因 |
|--------|--------|--------|----------|
| 用户登录方式 | 仅支持账号密码登录 | 新增微信、支付宝第三方登录 | 用户反馈登录方式单一 |
```

| 字段 | 说明 | 必填 |
|------|------|------|
| 变更项 | 具体变更的功能点或内容 | ✅ |
| 变更前 | 变更之前的内容 | ✅ |
| 变更后 | 变更之后的内容 | ✅ |
| 变更原因 | 为什么做这个变更 | ✅ |

### 完整示例

```markdown
## 2024-01-15

### 📄 电商平台_main_prd.md (v1.0 → v1.1)

**变更概述**：调整用户管理模块的功能需求

#### 第3章 功能模块清单

| 变更项 | 变更前 | 变更后 | 变更原因 |
|--------|--------|--------|----------|
| 用户登录方式 | 仅支持账号密码登录 | 新增微信、支付宝第三方登录 | 用户反馈登录方式单一，影响转化率 |
| 密码规则 | 6位以上任意字符 | 8位以上，必须包含大小写字母和数字 | 提升账户安全性 |

---

### 📄 modules/001_用户管理_prd.md (v1.0 → v1.1)

**变更概述**：优化登录失败提示

#### 第3章 详细功能需求

| 变更项 | 变更前 | 变更后 | 变更原因 |
|--------|--------|--------|----------|
| 登录失败提示 | 统一提示"登录失败" | 区分提示具体原因 | 提升用户体验 |
```

### 变更记录位置

- **CHANGELOG.md**：记录详细变更内容（变更前后对比表格）
- **PRD文档内**：记录版本概述
- **PRD_INDEX.md**：如果模块名称或版本变化，更新索引

### 详细格式规范

完整的 CHANGELOG 格式规范请参考：[templates/changelog_format.md](../templates/changelog_format.md)

## ⚠️ 格式要求

更新 PRD 时，必须：

- **严格遵循对应的模板文件格式**（`main_prd_template.md` 或 `module_prd_template.md`）
- **保持格式一致性**：与现有文档格式保持一致
- **保持章节结构**：不改变章节顺序和编号
- **完整替换变量**：所有 `{{变量名}}` 必须替换为实际内容
- **生成后验证**：检查格式是否符合模板要求

## 注意事项

- 支持单章节更新，无需重新生成整个文档
- 保持向后兼容
- 所有变更都会记录到 CHANGELOG.md
