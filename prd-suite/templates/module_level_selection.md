# XMind 层级选择功能说明

## 概述

在使用 XMind 文件创建主 PRD 时，AI 会自动分析 XMind 的层级结构，推荐最合适的功能模块提取层级，用户提供三种选择方式。

## 触发时机

在 XMind 内容解析完成后，用户确认 XMind 内容文档之后，进入层级选择流程。

## 三种选择方式

### 选项 1：确认 AI 推荐

**适用场景**：不确定选哪层，信任 AI 判断

**AI 推荐逻辑**：
| 评分维度 | 权重 | 评分标准 |
|----------|------|----------|
| 节点数量 | 30 分 | 5-12 个得满分，3-15 个得 20 分 |
| 标题长度 | 25 分 | 4-8 个字得满分 |
| 语义性 | 25 分 | 50% 以上包含功能模块关键词 |
| 层级深度 | 20 分 | 第 2 层得满分，第 1/3 层得 10 分 |

**功能模块关键词**：
管理、功能、模块、系统、中心、服务、配置、设置、用户、订单、支付、商品、库存、物流、营销、数据、分析、报表、统计、权限、角色、菜单、日志、消息

### 选项 2：按固定层级提取

**适用场景**：明确知道 XMind 结构，有特定需求

**操作流程**：
1. 用户输入层级数字（1-5）
2. AI 展示该层所有节点
3. 用户确认后提取

**输入示例**：
```
请输入层级数字（1-5）：
> 2

已选择：第 2 层（共 8 个节点）
提取的功能模块：
1. 用户管理
2. 订单管理
3. 支付管理
...

确认是否继续？[确认/重新选择]
```

### 选项 3：手动指定功能模块清单

**适用场景**：XMind 结构特殊，或需要完全自定义

**输入格式**：
```
编号 | 名称 | 描述 | 优先级
```

**输入示例**：
```
M-01 | 用户管理 | 负责用户相关功能，包括注册、登录、信息管理 | 高
M-02 | 订单管理 | 负责订单处理功能，包括下单、支付、发货 | 高
M-03 | 数据分析 | 负责数据统计和报表功能 | 中

输入完成后，发送"完成"结束：
> 完成
```

**解析规则**：
- 编号：可选，默认生成 M-01, M-02...
- 名称：必填，模块名称
- 描述：可选，模块简要描述
- 优先级：可选，高/中/低，默认"中"

## 层级分析展示格式

```
═════════════════════════════════════════════
📊 XMind 结构分析

根主题：{产品名称}

层级结构预览：
├── 第 1 层（共 X 个节点）
│   ├── 节点 1、节点 2、节点 3...
├── 第 2 层（共 X 个节点）
│   ├── 用户管理、订单管理、支付管理...
└── 第 3 层（共 X 个节点）
    ├── 用户列表、用户详情、创建用户...

🤖 AI 分析推荐
推荐层级：第 X 层
理由：
- 节点数量适中（X 个），符合模块划分粒度
- 节点命名清晰，符合功能模块命名规范
- 与主 PRD 功能模块清单的颗粒度匹配

推荐的功能模块清单：
1. 模块 1
2. 模块 2
3. 模块 3
...
═════════════════════════════════════════════

请选择功能模块的提取层级：
1️⃣ 确认使用 AI 推荐（第 X 层，X 个模块）
2️⃣ 按固定层级提取（输入层级数字 1/2/3/4/5）
3️⃣ 手动指定功能模块清单
```

## 异常处理

### 情况 1：XMind 层级不足
```
⚠️ 该 XMind 文件只有 X 层节点，无法选择第 Y 层。
请选择其他层级（1-X）或手动指定功能模块清单。
```

### 情况 2：某层节点数量过多
```
⚠️ 第 X 层包含 XX 个节点，数量较多，可能过于细化。
建议选择第 Y 层（XX 个节点）或手动指定功能模块清单。

确认是否继续？[继续/重新选择]
```

### 情况 3：某层节点数量过少
```
⚠️ 第 X 层只有 X 个节点，数量较少，可能过于粗略。
建议选择第 Y 层（XX 个节点）或手动指定功能模块清单。

确认是否继续？[继续/重新选择]
```

## 输出结果

层级选择完成后，输出以下结果用于后续 PRD 生成：

```json
{
  "selectedLevel": 2,
  "selectionMode": "ai_recommended",
  "modules": [
    {
      "id": "M-01",
      "name": "用户管理",
      "description": "",
      "priority": "中",
      "sourcePath": "根主题 > 核心功能 > 用户管理"
    },
    ...
  ]
}
```

**selectionMode 取值**：
- `ai_recommended` - AI 推荐
- `fixed_level` - 固定层级
- `manual` - 手动指定

## 与后续流程的衔接

层级选择完成后：
1. 将选定的功能模块清单填充到 `xmind_content.md` 的"2.4 功能模块列表"章节
2. 更新"内容分析与总结"部分
3. 进入步骤 8：提取 PRD 信息
4. 继续生成主 PRD

## 相关文件

- `commands/create.md` - 主流程文档（含层级选择步骤）
