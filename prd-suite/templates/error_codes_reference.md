# 错误码参考列表

## 概述

本文档提供了标准的错误码列表，用于模块 PRD 中错误码表的生成。所有错误码应从本列表中选择。

## 错误码分类

### 1. 成功状态码（1）

| 错误码 | 错误信息 | 说明 |
| --- | --- | --- |
| 1 | 操作成功 | 请求成功完成 |

### 2. 参数错误（10001-19999）

| 错误码 | 错误信息 | 说明 |
| --- | --- | --- |
| 10001 | 参数无效 | 参数格式或值不符合要求 |
| 10002 | 参数为空 | 必填参数未提供 |
| 10003 | 参数类型错误 | 参数类型不匹配 |
| 10004 | 参数缺失 | 缺少必要的参数 |

**使用场景：**
- API 请求参数验证失败
- 表单提交数据不完整
- 数据格式不符合要求

### 3. 用户错误（20001-29999）

| 错误码 | 错误信息 | 说明 |
| --- | --- | --- |
| 20001 | 用户未登录 | 用户未登录或登录已过期 |
| 20002 | 账号不存在或密码错误 | 登录时账号或密码错误 |
| 20003 | 账号已被禁用 | 用户账号被管理员禁用 |
| 20004 | 用户不存在 | 查询的用户不存在 |
| 20005 | 用户已存在 | 注册时用户名已被使用 |
| 20006 | 凭证已存在 | 登录凭证已存在 |
| 20007 | 用户已在别处登录，谨防密码泄露 | 同一账号在其他设备登录 |
| 20008 | 登录超时，请重新登录 | 登录会话超时 |
| 20009 | 账号已被锁定 | 用户账号被锁定 |
| 20010 | 账号已被删除 | 用户账号已被删除 |
| 20011 | 用户未关联可用组织 | 用户未关联任何组织 |
| 20012 | 用户未选择组织 | 用户需要选择组织 |
| 20013 | 首次登录或密码过期，需更新密码 | 需要修改密码 |

**使用场景：**
- 用户认证相关操作
- 用户信息管理
- 登录状态检查

### 4. 业务错误（30001-39999）

| 错误码 | 错误信息 | 说明 |
| --- | --- | --- |
| 30001 | 业务错误 | 通用业务逻辑错误 |

**使用场景：**
- 业务规则验证失败

## 错误码选择指南

### 按模块类型选择

#### 用户管理模块

常用错误码：
- 20001：用户未登录

- 50001：数据未找到

- 10001：参数无效

- 30002：关联引用,不允许删除

优先使用：90001-99999（文件错误）
常用错误码：
- 90001：文件上传异常

- 50001：数据未找到

## 错误信息生成规则

### 格式
```
{标准错误信息}：{具体场景描述}
```

### 示例

#### 标准错误信息
- 参数无效
- 用户未登录
- 数据未找到

#### 具体场景描述
- 用户ID不能为空
- 访问用户列表时未登录
- 查询用户详情时用户不存在

#### 最终错误信息
- 参数无效：用户ID不能为空
- 用户未登录：访问用户列表时未登录
- 数据未找到：查询用户详情时用户不存在

## 说明编写规则

### 内容要求
1. 描述错误发生的具体场景
2. 说明错误触发条件
3. 可选：提供解决方案

### 示例

#### 简单说明
- 创建用户时未提供用户ID
- 访问用户列表时未登录
- 查询用户详情时用户不存在

#### 详细说明
- 创建用户时未提供用户ID，请检查请求参数
- 访问用户列表时未登录，请先登录系统
- 查询用户详情时用户不存在，请确认用户ID是否正确

## 注意事项

### 1. 错误码唯一性
- 每个错误码在系统中应唯一
- 不要随意创建新的错误码
- 优先使用已有的标准错误码

### 2. 错误信息清晰性
- 错误信息应清晰明确
- 避免使用技术术语
- 提供足够的上下文信息

### 3. 错误码组合
- 一个模块可能涉及多种类型的错误
- 根据实际功能选择合适的错误码组合
- 不要过度使用错误码，保持简洁

### 4. 错误码扩展
- 如需新增错误码，应遵循现有分类规则
- 新增错误码应在对应分类的范围内
- 新增错误码应经过评审

## 总结

本错误码参考列表提供了标准的错误码定义和使用指南，确保：
- 📋 错误码统一规范
- 🎯 错误码选择有据可依
- 💡 错误信息清晰明确
- ✅ 便于前后端对接

所有模块 PRD 中的错误码表都应参考本列表生成。
