# PRD 审核确认流程说明

## 概述

本文档详细说明了 PRD 创建和更新过程中的审核确认流程，确保所有 PRD 文档在正式创建或更新前都经过用户的审核确认。

## 核心原则

### 1. 审核即写入
- 生成内容后展示给用户审核
- 审核通过后**直接写入文件**，不经过中间步骤
- 输出简短完成提示，不再次展示完整内容

### 2. 必须暂停等待
- AI 必须暂停等待用户审核确认
- 不得自动跳过审核步骤
- 不得在未审核的情况下继续执行

### 3. 支持迭代修改
- 用户可以多次提出修改意见
- AI 根据意见修改后再次展示审核
- 直到用户满意为止

### 4. 提供完整选项
- 必须提供确认写入、修改、重新生成、取消四个选项
- 每个选项都有明确的后续行为
- 用户可以根据实际情况选择

## 审核流程

### 优化后流程图

```
生成内容 → 展示内容 → 暂停等待审核 → 用户选择
                                    ↓
                    ┌───────────────┼───────────────┬─────────────┐
                    ↓               ↓               ↓             ↓
                确认写入         修改意见        重新生成         取消
                    ↓               ↓               ↓             ↓
              直接写入文件     修改内容后      重新生成后       终止流程
              输出简短提示     再次审核        再次审核
```

### 详细步骤

#### 步骤 1：生成内容
- AI 根据收集的信息生成 PRD 内容
- 确保内容符合模板格式要求

#### 步骤 2：展示内容
- 将生成的完整内容展示给用户查看
- 使用清晰的格式展示
- 突出重点内容

#### 步骤 3：暂停等待审核
- ⚠️ **必须暂停，等待用户审核确认**
- **不能自动继续执行后续操作**
- 明确告知用户需要审核确认

#### 步骤 4：提供审核选项
向用户提供以下四个选项：

##### ✅ 确认写入
- **含义**：用户确认内容无误
- **后续操作**：直接写入文件，输出简短完成提示
- **完成提示格式**：`✅ 主 PRD 已创建：prd/main_prd.md（约 XXX 字）`
- **使用场景**：内容完全符合预期，无需修改

##### ✏️ 提出修改意见
- **含义**：用户对部分内容有修改意见
- **后续操作**：
  1. 用户输入具体的修改意见
  2. AI 根据意见修改内容
  3. 修改后再次展示，进入新一轮审核
- **使用场景**：内容基本符合预期，但需要局部调整

##### 🔄 重新生成
- **含义**：用户对整体内容不满意
- **后续操作**：
  1. AI 重新生成内容
  2. 重新生成后再次展示审核
- **使用场景**：内容方向或整体结构不符合预期

##### ❌ 取消操作
- **含义**：用户取消本次操作
- **后续操作**：
  1. 终止流程
  2. 不创建或更新任何文件
- **使用场景**：不再需要创建或更新 PRD

## 核心优化说明

### 一次展示原则
- 确认写入后**直接写文件**，不再二次展示内容
- 避免"审核 → 确认 → 再展示 → 写入"的冗余流程

### 简短完成提示
- 完成时只输出文件路径和字数
- 不输出完整内容，减少 token 消耗

### 修改时才重展示
- 只有用户提出修改意见时，才重新展示修改后的内容
- 确认写入不触发二次展示

## 审核选项使用指南

### 何时选择"确认写入"

- ✅ 内容完全符合预期
- ✅ 格式正确，符合模板要求
- ✅ 所有信息准确无误
- ✅ 没有遗漏关键内容

### 何时选择"提出修改意见"

- ✏️ 内容基本符合预期，但需要局部调整
- ✏️ 某些描述不够准确或清晰
- ✏️ 需要补充或删除某些内容
- ✏️ 格式有小的瑕疵需要修正

**修改意见示例：**

1. **内容修改**
   ```
   请将"用户管理模块"改为"用户权限管理模块"
   ```

2. **描述优化**
   ```
   在"背景说明"中，请补充市场竞争分析的内容
   ```

3. **格式调整**
   ```
   请将表格中的"优先级"列移到"描述"列之前
   ```

4. **内容补充**
   ```
   在"非功能需求"中，请添加"数据备份"相关要求
   ```

### 何时选择"重新生成"

- 🔄 整体方向不符合预期
- 🔄 章节结构需要大幅调整
- 🔄 内容质量明显不符合要求
- 🔄 多次修改后仍不满意

### 何时选择"取消操作"

- ❌ 不再需要创建或更新 PRD
- ❌ 项目需求发生重大变化，需要重新规划
- ❌ 发现信息收集阶段有重大遗漏
- ❌ 其他原因导致无法继续

## 常见问题

### Q1: 如果多次修改后仍不满意怎么办？

**A:** 可以选择"重新生成"或"取消操作"：
- 选择"重新生成"：AI 会重新生成内容，可能得到更好的结果
- 选择"取消操作"：终止流程，重新规划需求

### Q2: 审核时发现信息收集阶段有遗漏怎么办？

**A:** 可以选择"取消操作"，然后重新开始流程，在信息收集阶段补充遗漏的信息。

### Q3: 可以只修改部分章节吗？

**A:** 可以。在更新 PRD 时，可以选择只更新特定章节，审核时也只审核该章节的内容。

### Q4: 审核通过后还能修改吗？

**A:** 可以。审核通过并创建文件后，如果需要修改，可以使用"update"命令更新相应章节，同样会经过审核确认流程。

### Q5: 确认写入后会再次展示内容吗？

**A:** 不会。确认写入后直接创建文件，只输出简短的完成提示（文件路径和字数），不再展示完整内容，避免冗余消耗。

## 总结

审核确认流程是确保 PRD 质量的关键环节：

- ✅ **必须暂停等待**：AI 不得自动跳过审核
- ✅ **提供完整选项**：确认写入、修改、重新生成、取消
- ✅ **支持迭代修改**：用户可多次提出修改意见
- ✅ **审核即写入**：确认后直接写入文件，不二次展示

遵循本流程可以确保：
- PRD 内容符合预期
- 用户有完全的掌控权
- 减少 token 冗余消耗
- 提升 PRD 质量
