# PRD 审核确认流程说明

## 概述

本文档详细说明了 PRD 创建和更新过程中的审核确认流程，确保所有 PRD 文档在正式创建或更新前都经过用户的审核确认。

## 核心原则

### 1. 先审核后执行
- 生成内容后必须先展示给用户审核
- 审核通过前不得创建或更新文件
- 确保用户对内容有完全的掌控

### 2. 必须暂停等待
- AI 必须暂停等待用户审核确认
- 不得自动跳过审核步骤
- 不得在未审核的情况下继续执行

### 3. 支持迭代修改
- 用户可以多次提出修改意见
- AI 根据意见修改后再次展示审核
- 直到用户满意为止

### 4. 提供完整选项
- 必须提供确认、修改、重新生成、取消四个选项
- 每个选项都有明确的后续行为
- 用户可以根据实际情况选择

## 审核流程

### 标准流程图

```
生成内容 → 展示内容 → 暂停等待审核 → 用户选择 → 执行相应操作
                                    ↓
                    ┌───────────────┼───────────────┬─────────────┐
                    ↓               ↓               ↓             ↓
                  确认           修改意见        重新生成         取消
                    ↓               ↓               ↓             ↓
                执行后续      修改内容后        重新生成后       终止流程
                操作          再次审核          再次审核
```

### 详细步骤

#### 步骤 1：生成内容
- AI 根据收集的信息生成 PRD 内容
- ⚠️ **此时不创建或更新文件，只生成内容**
- 确保内容符合模板格式要求

#### 步骤 2：展示内容
- 将生成的完整内容展示给用户查看
- 使用清晰的格式展示
- 突出重点内容

#### 步骤 3：暂停等待审核
- ⚠️ **必须暂停，等待用户审核确认**
- **不能自动继续执行后续操作**
- 明确告知用户需要审核确认

#### 步骤 4：提供审核选项
向用户提供以下四个选项：

##### ✅ 确认通过
- **含义**：用户确认内容无误
- **后续操作**：继续执行后续操作（创建文件、更新索引等）
- **使用场景**：内容完全符合预期，无需修改

##### ✏️ 提出修改意见
- **含义**：用户对部分内容有修改意见
- **后续操作**：
  1. 用户输入具体的修改意见
  2. AI 根据意见修改内容
  3. 修改后再次展示，进入新一轮审核
- **使用场景**：内容基本符合预期，但需要局部调整

##### 🔄 重新生成
- **含义**：用户对整体内容不满意
- **后续操作**：
  1. AI 重新生成内容
  2. 重新生成后再次展示审核
- **使用场景**：内容方向或整体结构不符合预期

##### ❌ 取消操作
- **含义**：用户取消本次操作
- **后续操作**：
  1. 终止流程
  2. 不创建或更新任何文件
- **使用场景**：不再需要创建或更新 PRD

#### 步骤 5：执行后续操作
- 只有在用户选择"确认通过"后才执行
- 创建或更新文件
- 更新相关索引和变更日志

## 审核选项使用指南

### 何时选择"确认通过"

- ✅ 内容完全符合预期
- ✅ 格式正确，符合模板要求
- ✅ 所有信息准确无误
- ✅ 没有遗漏关键内容

### 何时选择"提出修改意见"

- ✏️ 内容基本符合预期，但需要局部调整
- ✏️ 某些描述不够准确或清晰
- ✏️ 需要补充或删除某些内容
- ✏️ 格式有小的瑕疵需要修正

**修改意见示例：**

1. **内容修改**
   ```
   请将"用户管理模块"改为"用户权限管理模块"
   ```

2. **描述优化**
   ```
   在"背景说明"中，请补充市场竞争分析的内容
   ```

3. **格式调整**
   ```
   请将表格中的"优先级"列移到"描述"列之前
   ```

4. **内容补充**
   ```
   在"非功能需求"中，请添加"数据备份"相关要求
   ```

5. **内容删除**
   ```
   请删除"附录"章节中的"参考资料2"
   ```

### 何时选择"重新生成"

- 🔄 整体方向不符合预期
- 🔄 章节结构需要大幅调整
- 🔄 内容质量明显不符合要求
- 🔄 多次修改后仍不满意

**重新生成提示示例：**

1. **方向调整**
   ```
   请重新生成，这次重点关注移动端用户体验
   ```

2. **结构调整**
   ```
   请重新生成，将"用户故事"章节提前到"功能需求"之前
   ```

3. **质量提升**
   ```
   请重新生成，提供更详细的功能描述和用户故事
   ```

### 何时选择"取消操作"

- ❌ 不再需要创建或更新 PRD
- ❌ 项目需求发生重大变化，需要重新规划
- ❌ 发现信息收集阶段有重大遗漏
- ❌ 其他原因导致无法继续

## 审核最佳实践

### 对于创建 PRD

#### 创建主 PRD
1. **审核重点**
   - 项目背景和目标是否清晰
   - 核心业务流程是否完整
   - 功能模块划分是否合理
   - 用户角色和权限是否准确

2. **常见修改意见**
   - 补充市场背景分析
   - 调整功能模块优先级
   - 细化用户故事描述
   - 完善非功能需求

#### 创建模块 PRD
1. **审核重点**
   - 模块目标是否与主 PRD 一致
   - 功能需求是否详细完整
   - 用户界面设计是否合理
   - 与其他模块的集成是否清晰

2. **常见修改意见**
   - 补充功能详细说明
   - 细化用户故事验收标准
   - 完善界面交互细节
   - 补充异常处理逻辑

### 对于更新 PRD

#### 更新主 PRD
1. **审核重点**
   - 更新内容是否准确
   - 是否影响其他章节
   - 版本管理是否正确
   - 变更记录是否完整

2. **常见修改意见**
   - 调整变更描述
   - 补充变更影响分析
   - 更新相关章节内容
   - 完善变更记录

#### 更新模块 PRD
1. **审核重点**
   - 更新内容是否准确
   - 是否影响其他模块
   - 集成关系是否需要调整
   - 验收标准是否需要更新

2. **常见修改意见**
   - 细化功能变更描述
   - 更新集成关系
   - 调整验收标准
   - 补充测试场景

## 常见问题

### Q1: 如果多次修改后仍不满意怎么办？

**A:** 可以选择"重新生成"或"取消操作"：
- 选择"重新生成"：AI 会重新生成内容，可能得到更好的结果
- 选择"取消操作"：终止流程，重新规划需求

### Q2: 审核时发现信息收集阶段有遗漏怎么办？

**A:** 可以选择"取消操作"，然后重新开始流程，在信息收集阶段补充遗漏的信息。

### Q3: 可以只修改部分章节吗？

**A:** 可以。在更新 PRD 时，可以选择只更新特定章节，审核时也只审核该章节的内容。

### Q4: 审核通过后还能修改吗？

**A:** 可以。审核通过并创建文件后，如果需要修改，可以使用"update"命令更新相应章节，同样会经过审核确认流程。

### Q5: 如果 AI 没有暂停等待审核怎么办？

**A:** 这是异常情况。请立即告知 AI 停止执行，并要求其遵循审核确认流程。AI 应该：
1. 生成内容后暂停
2. 展示内容
3. 明确询问用户审核意见
4. 等待用户选择后再继续

## 总结

审核确认流程是确保 PRD 质量的关键环节：

- ✅ **必须暂停等待**：AI 不得自动跳过审核
- ✅ **提供完整选项**：确认、修改、重新生成、取消
- ✅ **支持迭代修改**：用户可多次提出修改意见
- ✅ **先审核后执行**：审核通过前不得创建或更新文件

遵循本流程可以确保：
- PRD 内容符合预期
- 用户有完全的掌控权
- 避免不必要的返工
- 提升 PRD 质量
