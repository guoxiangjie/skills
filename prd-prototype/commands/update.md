# update 子命令

## 功能说明

根据检查结果，增量更新原型页面，保持原型与 PRD 同步。

## 前置条件

- 原型目录 `prd/prototype/` 必须存在
- 映射文档 `prd/prototype/mapping.md` 必须存在

## 工作流程

### 步骤 1：验证映射文档存在

检查 `prd/prototype/mapping.md` 是否存在。

如果不存在：
> ⚠️ 未找到映射文档。请先执行 `/prd-prototype generate` 生成原型。

### 步骤 2：执行同步检查

自动执行 check 命令的逻辑，识别：

1. **需更新的页面** - 依赖的 PRD 版本已变更
2. **新增的 PRD** - 新增模块未生成原型
3. **删除的 PRD** - PRD 已删除但原型页面存在

### 步骤 3：展示更新清单

```
📋 原型更新清单

【需更新页面】
├── user-management.html
│   └── 原因：M001_用户管理_prd.md v1.0 → v1.1
├── order-management.html
│   └── 原因：M001_用户管理_prd.md 版本变更影响
└── payment-management.html
    └── 原因：M003_支付管理_prd.md v1.0 → v1.2

【需新增页面】
└── stock-management.html（库存管理）
    └── 原因：新增 M004_库存管理_prd.md

【需删除页面】
└── report-management.html
    └── 原因：M005_报表管理_prd.md 已删除

─────────────────────────
更新统计：更新 3 页、新增 1 页、删除 1 页
```

### 步骤 4：用户确认更新范围

使用 AskUserQuestion 工具询问用户：

```
请选择更新范围：

1. 更新全部（更新 3 页 + 新增 1 页 + 删除 1 页）
2. 仅更新现有页面（更新 3 页）
3. 仅新增页面（新增 1 页）
4. 手动选择更新项
5. 取消
```

### 步骤 5：读取 PRD 变更内容

对每个需更新的页面：

1. 读取对应的 PRD 文件
2. 对比 mapping.md 中记录的依赖章节
3. 提取变更内容

### 步骤 6：增量更新页面

#### 更新现有页面

```
页面：user-management.html

1. 读取当前页面内容
2. 读取 M001_用户管理_prd.md 最新内容
3. 分析变更影响：
   - 第3章新增批量删除功能 → 表格操作列新增批量删除按钮
   - 第4章用户ID字段长度变更 → 无界面影响（仅后端校验）
4. 重新生成受影响的部分
5. 保留未变更的部分
```

#### 新增页面

```
页面：stock-management.html

1. 读取 M004_库存管理_prd.md
2. 按生成流程创建新页面
3. 更新导航菜单（所有页面的侧边栏）
4. 添加到 mapping.md
```

#### 删除页面

```
页面：report-management.html

1. 删除页面文件
2. 更新其他页面的导航菜单（移除入口）
3. 从 mapping.md 移除记录
```

### 步骤 7：更新映射文档

更新 `prd/prototype/mapping.md`：

1. **更新页面依赖映射** - 更新版本号、依赖内容
2. **更新 PRD 版本快照** - 更新各 PRD 版本
3. **追加更新历史** - 记录本次更新操作

```markdown
## 更新历史

| 更新时间 | 操作 | 更新页面 | 变更说明 |
|----------|------|----------|----------|
| 2026-04-03 10:30:00 | 增量更新 | user-management.html | M001 v1.0→v1.1，新增批量删除 |
| 2026-04-03 10:30:00 | 增量更新 | order-management.html | M001 版本变更影响 |
| 2026-04-03 10:30:00 | 新增页面 | stock-management.html | 新增库存管理模块 |
| 2026-04-03 10:30:00 | 删除页面 | report-management.html | PRD 已删除 |
```

### 步骤 8：更新 CHANGELOG

如果 `prd/CHANGELOG.md` 存在，追加更新记录：

```markdown
## {{CURRENT_DATE}}

### 📄 prd/prototype/ 🟡更新

原型增量更新，同步 PRD 版本变更。

#### 变更章节：页面文件

**变更内容**：更新 user-management.html、order-management.html，新增 stock-management.html，删除 report-management.html。
```

### 步骤 9：完成确认

```
✅ 原型更新完成

更新结果：
├── 更新页面：3 个
├── 新增页面：1 个
├── 删除页面：1 个
└── 更新 mapping.md

下一步：
- 执行 /prd-prototype preview 预览更新后的原型
- 执行 /prd-prototype check 确认同步状态
```

## 更新策略

### 部分更新 vs 完整重新生成

| 场景 | 策略 |
|------|------|
| 单个章节内容变更 | 部分更新受影响区域 |
| 多个章节变更 | 重新生成整个页面 |
| 设计体系变更 | 重新生成全部页面 |
| 技术栈变更 | 重新生成全部页面 |

### 导航菜单同步

当新增或删除页面时，需要同步更新所有现有页面的侧边栏导航菜单。

## 注意事项

- 更新前建议备份原型目录
- 更新后会保留用户的自定义修改（如果不在变更范围内）
- 如果 PRD 结构发生重大变化，建议执行 `generate --full` 重新生成
