# check 子命令

## 功能说明

检查原型页面与 PRD 文档的同步状态，发现版本差异，输出检查报告。

## 前置条件

- 原型目录 `prd/prototype/` 必须存在
- 映射文档 `prd/prototype/mapping.md` 必须存在

## 工作流程

### 步骤 1：验证映射文档存在

检查 `prd/prototype/mapping.md` 是否存在。

如果不存在：
> ⚠️ 未找到映射文档。请先执行 `/prd-prototype generate` 生成原型。

### 步骤 2：读取映射文档

解析 mapping.md，提取：

1. **页面依赖映射** - 每个页面依赖的 PRD 文件、版本、章节
2. **PRD 版本快照** - 记录的所有 PRD 文件及版本

### 步骤 3：读取 PRD 当前版本

遍历映射文档中记录的所有 PRD 文件：
- 读取 `prd/main_prd.md`，提取第 1 章的"当前版本"字段
- 读取 `prd/modules/*.md`，提取各模块 PRD 的"当前版本"字段

### 步骤 4：对比版本差异

对每个页面检查其依赖的所有 PRD：

```
页面: user-management.html
├── 依赖 M001_用户管理_prd.md v1.0 → 当前 v1.1 ⚠️ 需更新
├── 依赖 main_prd.md v1.0 → 当前 v1.0 ✅ 同步
└── 结论: ⚠️ 需更新
```

### 步骤 5：生成检查报告

输出检查报告：

```markdown
📋 原型同步检查报告

**检查时间**：{{YYYY-MM-DD HH:mm:ss}}

## 同步状态总览

| 状态 | 数量 |
|------|------|
| ✅ 同步 | {{N}} |
| ⚠️ 需更新 | {{N}} |
| ❌ PRD 文件缺失 | {{N}} |
| 🆕 新增 PRD 未生成原型 | {{N}} |

---

## 页面同步状态

| 页面 | 依赖 PRD 数量 | 状态 | 说明 |
|------|---------------|------|------|
| index.html | 2 | ✅ 同步 | - |
| dashboard.html | 1 | ✅ 同步 | - |
| user-management.html | 2 | ⚠️ 需更新 | M001_用户管理_prd.md v1.0 → v1.1 |
| order-management.html | 2 | ⚠️ 需更新 | M001_用户管理_prd.md 版本变更 |

---

## 变更详情

### ⚠️ user-management.html

| 依赖 PRD | 记录版本 | 当前版本 | 影响 |
|----------|----------|----------|------|
| M001_用户管理_prd.md | v1.0 | v1.1 | 第3章新增批量删除功能，第4章用户ID字段长度变更 |

**建议操作**：重新生成此页面

---

### ⚠️ order-management.html

| 依赖 PRD | 记录版本 | 当前版本 | 影响 |
|----------|----------|----------|------|
| M001_用户管理_prd.md | v1.0 | v1.1 | 用户ID字段长度变更，影响订单列表用户信息展示 |

**建议操作**：更新用户信息展示部分

---

## 新增 PRD（未生成原型）

| PRD 文件 | 当前版本 | 说明 |
|----------|----------|------|
| M004_库存管理_prd.md | v1.0 | 新增模块，建议生成原型 |

---

## 建议操作

执行以下命令更新原型：

```
/prd-prototype update
```

将更新以下页面：
- user-management.html
- order-management.html

将新增以下页面：
- stock-management.html（库存管理）
```

### 步骤 6：检查新增 PRD

扫描 `prd/modules/` 目录，检查是否有新增的模块 PRD 未在 mapping.md 中记录。

如有，在报告中列出"新增 PRD 未生成原型"。

### 步骤 7：输出结论

```
📋 检查完成

同步状态：
├── ✅ 同步：2 个页面
├── ⚠️ 需更新：2 个页面
└── 🆕 新增 PRD：1 个模块

下一步：
- 执行 /prd-prototype update 更新原型
- 或执行 /prd-prototype generate --full 重新生成全部原型
```

## 注意事项

- 检查过程中不会修改任何文件
- PRD 版本从文档第 1 章的"当前版本"字段读取
- 如果 PRD 文件被删除，会在报告中标记为"PRD 文件缺失"
